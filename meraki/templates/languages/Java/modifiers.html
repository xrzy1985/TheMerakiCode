Java's Modifiers information:

Modifiers are keywords, added to declarations, to change properties of a method or variable. The Java language supports the following modifiers, in addition to others not expounded here:

  • "Access" Modifiers

  • "Non-Access" Modifiers

To use a modifier, simply add the keyword for it to the definition of a class, method, or variable. The modifier should be the first part of the statement, as shown below:

Example

public class className {
   // ...
}

private boolean myFlag;
static final double weeks = 9.5;
protected static final int BOXWIDTH = 42;

public static void main(String[] arguments) {
   // body of method
}

Access Control Modifiers

Java has several access modifiers available. They set access allowance for classes, variables, and methods. The four access levels are:

  • Only visible to the package (Default accessibility. No modifiers are needed.)
  • Only visible to the class (private keyword used).
  • Publicly visible to all other packages, classes, and methods (public keyword used).
  • Only visible within the package and subclasses of the class (protected keyword used).

Non-Access Modifiers

Java also provides non-access modifiers, which allow other functionality.

  • The modifier static allows us to create class methods and variables.

  • The modifier final allows us to finalize the implementations of classes, methods, and variables.

  • The modifier abstract allows the creation of abstract classes and methods.

  • The modifierssynchronized and volatile are used in threaded processes.
