Java's Data Structures information:

The Java Uitility package includes a host of complex, incredibly useful data structures. Some of the most common are detailed here for this tutorial.

These data structures consist of the following interface and classes −

  • Enumeration
  • BitSet
  • Vector
  • Stack
  • Dictionary
  • Hashtable
  • Properties

The Enumeration

The Enumeration interface is not, itself, an instantiatable data structure, but it provides methods that other classes can extend it to utilize, allowing them a means to retrieve successive elements from a data structure.

Enumeration, for example, contains the method nextElement, which is used to get the next element in a data structure.

The BitSet

The BitSet class implements a group of bits (flags) that can be toggled or cleared.

This class is useful for cases where multiple boolean values are required, as they can easily be set to new values or checked for their current value.

The Vector

The Vector class is similar in function to an Array in Java, though it is not of a fixed length like a Java Array is.

Vectors, like Arrays, allow elements to be accessed via the element's index.

Vectors can be easier to work with than Arrays at times, as the size doesn't need to be manually set at creation and it doesn't have a set size to restrict your usage of it.

The Stack

The Stack class implements a "stack" of elements.

A stack operates literally like a stack of physical objects would. Objects added to the stack are placed "on top", and objects removed are removed "from the top".

Because of this, the last elements added to a stack are the first to be removed/accessed.

The Dictionary

The Dictionary class is abstract, so other classes extend it to utilize its functions. Its functions allow a class to create a data structure for mapping "keys" to "values".

This is implemented in the Hashtable class, which uses it.

The Hashtable

The Hashtable class allows you to store data in a key/value structure, as described in the dictionary.

A hashtable allows many types of data to be the key or value. For example, the key could be a string value and the value could be an integer. In that case, the Hashtable would be described with Hashtable<String,int> while creating it, like in this example:

Hashtable<String,int> myHashtable = new Hashtable<String,int>;

That example creates a new hashtable that can accept (via the .put(String,int) command) any String and integer combination, with the String value serving as a key used to reference/check the value of the integer value.

The Properties

Properties is a subclass of Hashtable. It is used to maintain lists of values in which the key is a String and the value is also a String.

The Properties class is utlized by some other Java classes. For example, System.getProperties( ) returns a Properties object.
