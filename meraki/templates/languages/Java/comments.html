Java's Comments and Their importance:
The Java language supports single-line and multi-line comments, in a fashion very similar to C and C++. Anything inside any comment is ignored by the Java compiler.

Comments are essential for any code that will be shared with others, worked on collaboratively, or even referenced/worked on by you in the future.
This is to say that comments are nearly always important. They allow you to describe, in the midst of the code, the things they code may be doing. The compiler ignores the comment entries and doesn't run it as code.

Java's comment structure is identical to C++, so the same examples as the C++ page on comments are included here:
I will first demonstrate how single line comments are done.


Single Line Comments

//I'm a single line comment!

For Multi Line comments, you start with /* and terminate the comment section with */

Multi Line Comments

/*This demonstrates
Multi Line Comments. */



Again, I will use the same Java file from the syntax page to give you a perfect example of how to properly comment code out. Comments are not stressed enough within programming classes, videos, or by professors. A good programmer can relearn what he/she wrote a few years ago, but a smart programmer doesn't have to remember because he/she wrote out effective comments within the code for themselves or other people who might read the code later on. Comments should be used in any fashion that is suitable, or in any way your company or professor directs you to. I prefer to code with single line comments, I don't have to close off them at the end, and it helps me keep things organized better. Find your own niche and run with it, but do get in the habit of commenting the code out properly.
