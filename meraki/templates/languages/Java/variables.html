Java's Variables information:
In Jaava, variables allow us to store values in a way that our methods can interact with. Each variable has a defined type, which determines: the allocated size of the variable's memory; the type of values that memory can store; and the operations you can use that value in.

Variables must be "declared" (created) before they can be assigned values or otherwise used. The following is a generic example of variable declaration:

dataType myVariable = ValueOfVariable;

Here dataType is generic representation of one of Java's data types (like an integer or string) and myVariable is the name of the variable. To declare multiple of the same type of variable in one line, you can use a comma-separated list.

The following are a few more examples of declaring variables in Java:

Examples

int a, b, c;         // Declares three ints, a, b, and c.
int a = 10, b = 10;  // Example of initialization
byte B = 22;         // initializes a byte type variable B.
double pi = 3.14159; // declares and assigns a value of PI.
char a = 'a';        // the char variable a iis initialized with value 'a'

This chapter will explain various variable types available in Java Language. There are three kinds of variables in Java −

  • Local variables
  • Instance variables
  • Class/Static variables

Local Variables

  • Local variables are declared in methods, constructors, or blocks.

  • Local variables are created when the method, constructor or block is entered and the variable will be destroyed once it exits the method, constructor, or block.

  • Access modifiers cannot be used for local variables.

  • Local variables are visible only within the declared method, constructor, or block.

  • Local variables are implemented at stack level internally.

  • There is no default value for local variables, so local variables should be declared and an initial value should be assigned before the first use.

Example

Here, age is a local variable. This is defined inside geckoAge() method and its scope is limited to only this method.

public class Test {
   public void geckoAge() {
      int age = 0;
      age = age + 7;
      System.out.println("Gecko's age is : " + age);
   }

   public static void main(String args[]) {
      Test test = new Test();
      test.geckoAge();
   }
}

This will produce the following result −

Output

Gecko's age is: 7

Example

Instance Variables

  • Instance variables are declared in a class, but outside of its methods/functions.

  • When a space is allocated for an object in the heap, a slot for each instance variable value is created.

  • Instance variables are created when an object is created with the use of the keyword 'new' and destroyed when the object is destroyed.

  • Instance variables are generally used for values that are referenced by more than one method, though an instance variable could be used by only one method.

  • Access modifiers, like public or private, can specify the access scope of instance variables.

  • Instance variables are visible for all methods/functions in the instance of the class/object. Instance variables are generally set as "private" to the class, unless a subclass needs access to them.

  • Instance variables, like all variables, have default values. The default value for ints is 0, for Booleans it is false, and for object references it is null. Variables can be assigned values in the declaration of a variable or within the variable's constructor constructor.

  • Instance variables can be accessed within the class by simply calling the variable by its name. From outside the class, though, (assuming public access to the variable), the fully qualified name for the variable should be used, ObjectReference.VariableName.

Example

import java.io.*;
public class Employee {

   // this instance variable is visible for any child class.
   public String name;

   // salary  variable is visible in Employee class only.
   private double salary;

   // The name variable is assigned in the constructor.
   public Employee (String empName) {
      name = empName;
   }

   // The salary variable is assigned a value.
   public void setSalary(double empSal) {
      salary = empSal;
   }

   // This method prints the employee details.
   public void printEmp() {
      System.out.println("name  : " + name );
      System.out.println("salary :" + salary);
   }

   public static void main(String args[]) {
      Employee empOne = new Employee("Joshua");
      empOne.setSalary(35000);
      empOne.printEmp();
   }
}

This will produce the following result −

Output

name  : Joshua
salary :35000.0

Class/Static Variables

  • Static variables (also called "Class Variables") are declared outside of any method/constructor and use the "static" keywork.

  • If a variable is static, there is only one instance of the variable and all instances of the class reference and use the same variable.

  • Scope of accessibility is similar to instance variables. However, most static variables are declared public since they must be available for users of the class.

  • Assuming the class calling them has access to them, Static variables can be accessed by calling with the class name ClassName.VariableName.

  • When declaring class variables as public static final, Java naming conventions recommend making the name of the variable all upper case. Otherwise, regular variable naming conventions are used.

Example

import java.io.*;
public class Employee {

   // salary  variable is a private static variable
   private static double salary;

   // DEPARTMENT is a constant
   public static final String DEPARTMENT = "Development ";

   public static void main(String args[]) {
      salary = 1000;
      System.out.println(DEPARTMENT + "average salary:" + salary);
   }
}

This will produce the following result −

Output

Development average salary:1000

Note − If the variables are accessed from an outside class, the constant should be accessed as Employee.DEPARTMENT
