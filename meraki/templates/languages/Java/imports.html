The import Keyword

For a class to reference a class in the same package, it can do so without importing it.

Example

In the following example, a class name Boss references another class Employee that is in the same package ("payroll"). The Boss class needs no special syntax or import to do so.

package payroll;
public class Boss {
   public void payEmployee(Employee e) {
      e.mailCheck();
   }
}

This would fail, however, if the Employee class was not in the same package. One of the following techniques would be necessary.

  • The Boss class could use the "fully qualified" title for it.
payroll.Employee

  • The boss class could import the entire package, using a wildcard.

import payroll.*;
  • The boss class could import just the Employee class, without a wildcard.
import payroll.Employee;

Note A class can important an unlimited number of packages, but be careful not to import large packages when only a class is needed or packages that are not needed. Each imported package increases the size and memory footprint of your program.
