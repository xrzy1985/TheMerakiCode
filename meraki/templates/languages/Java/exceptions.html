Java's Exceptions information:

This page deals with common exceptions in Java, as well as the basics of dealing with exceptions.

Exceptions are "events" when problems occur during a program's execution. An uncaught exception interrupts the flow of a program and causes it to close, which is generally not the desired behavior of a program. Because of that, it is important to catch and appropriately handle exceptions wherever they are likely to occur in your program.

There are many reasons that exceptions occur. Some of the following are common exceptions:

  • A user has entered an invalid data.

  • A file that needs to be opened cannot be found.

  • A network connection has been lost in the middle of communications or the JVM has run out of memory.

Some of these exceptions are due to: programmer errors, user errors, or failed resource access.

These three reasons form the basis for each of the three categories of exceptions in Java.

  • Checked exceptions − Checked exceptions occur at compile time, so they are also called compile time exceptions. These exceptions prevent the program from being compiled, so must be dealt with before it is compiled or run.

For example, if you use FileReader class in your program to read data from a file, if the file specified in its constructor doesn't exist, then a FileNotFoundException occurs, and the compiler prompts the programmer to handle the exception.

Catching Exceptions

Catching an exception is a method of dealing with exceptions. This is done using a combination of the try and catch keywords. A "try/catch block" is surrounds the code that may throw an exception to handle any exceptions it throws. This code surrounded by a try/catch is called "protected code", and the syntax for using try/catch looks like the following −

Syntax

try {
   // Protected code
}catch(ExceptionName e1) {
   // Catch block
}

The code that may throw the exception is placed in the try block of the try/catch. This is the code the compiler runs like usual.

The catch statement (or catch statements, plural) follow directly after thee try block. These statments catch specific exceptions thrown by the try block and run the code they contain. You should be sure to add code that addresses the exception to these blocks. If no appropriate catch block exists for the exception, the "finally" block is run instead, if present.

The Throws/Throw Keywords

If a method does not handle an exception itself, then it must throw the exception instead. The keyword throws, along with the exception it throws, is written at the end of the method signature.

The throw keyword is used to throw exceptions.

The example below shows a method that throws the RemoteException:

import java.io.*;
public class className {

   public void deposit(double amount) throws RemoteException {
      // Method implementation
      throw new RemoteException();
   }
   // Remainder of class definition
}
