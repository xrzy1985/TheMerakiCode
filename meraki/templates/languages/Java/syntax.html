Essentials of Java Syntax

Java is an amazing high level object oriented language. Java is designed around the idea of object oriented programming. In Java, everything can be considered an object. Java is a class based programming language; every Java file created is effectively a new class, but interfaces, abstract classes, and a few other exceptions do apply. Java was designed to allow a programming to write the code, and then be able to run the code on any machine. Java code is broken into java byte code upon compilation, and the java virtual machine located on the host computer will break the java byte code down into machine specific code. Java use to hold the reputation for being slower than C++ and other competitors, but since 1997 the company has made vast improvements on the speed of compilation.

Java was influenced heavily by C++. Java code is written all inside of classes with every item being an object. Java will not allow multiple inheritance for classes, but does offer interfaces and abstract classes. Java has a similar commenting system as C++.

// is a single line comment

/* will start a long comment, but it must be closed with a */ at the end in order to close off the comment.

Javadoc commenting style opened with /** and closed with */. The Javadoc style of commenting allows the user to run the Javadoc executable to create documentation for the program, and can be read by some integrated development environments (IDEs) such as Eclipse to allow developers to access documentation within the IDE.


  • Object − Objects have behaviours and states. For example, a gecko has "states" (traits like color, size, morphism) as well as "behaviours" (actions like eating, climbing, or escaping).

  • Class − A class can be described as a template/design that defines an object, as well as its states/behaviours.

  • Methods − A Java class's methods define its possible behaviour. There is virtually no limit to the number of methods your class can contain.

  • Instance Variables − Each class also has instance variables that describe its current state.

Basic Syntax

About Java programs, it is very important to keep in mind the following points.

  • Case Sensitivity − Java is case sensitive, which means the identifier names Hello and hello would be interepreted as different names by Java

  • Class Names − In standard practice, all class names should start with a capital letter. If the name is multiple words, "Camel Case", where the first letter of each word is capitalized, should be used.

    Example: class HelloWorld.java

  • Method Names − Mmethod names should generally start with a Lower Case letter. CamelCase should be used for every word but the first word in a longer name.

    Example: public void getName()

  • Program File Name − The name of a java program should match the name of the class.

    When saving a program, you should save it using the class name (while remembering that Java is case sensitive) and append '.java' to the end of the name (if the file name and the class name do not match, your program will not compile).

    Example: Assume 'MyFirstCoolJavaProgram' is the class name. Then the file should be saved as 'MyFirstCoolJavaProgram.java'

  • public static void main(String args[]) − Java program processing starts from the main() method which is a mandatory part of every Java program.
