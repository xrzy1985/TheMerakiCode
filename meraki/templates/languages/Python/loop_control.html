Controlling a loop is vital to programming especially in Python. Statements are typically read line by line as the interpreter runs through the code, but a loop can ensure your structure is followed properly. That means the first statement is read, the second is read, and so on until the end of the code or an error occurs. There might be a time or two when you need to run a specific block of code more than once, or call a function recursively. Loops allow the programmer the ability to add additional paths to be executed. A loop allows the programmer to access a block of code once, twice, or however many times they wish.




Python provides While, For, and Nested loops for programmers to use


  • While
    • -- the loop will run until the condition returns False, the condition is tested before the loop
  • For
    • -- will execute the loop block a certain number of times, abbreviates the code that handles the loop variable
  • Nested
    • -- you can nest loops within loops; while inside for, for inside for, for inside while, etc

The loop control property will change the way the python file is executed or run. The sequence in which the file is read will be altered, slightly. Any variables or data created while inside a loop will be destroy upon exiting the specific block of code. So, if you declare a variable inside a for loop, that variable will be destroyed in memory once the code block has been exited. The programmer has four main options in a loop; allow the loop to run unaltered, break the loop using a control statement, continue, or pass.



Break Control Statements

Terminates the loop statement and transfers execution to the statement immediately following the loop

break

Continue Control Statements

Causes the loop to skip the remainder of its body and immediately retest its condition prior to reiterating.

continue

Pass Control Statements

The pass statement in Python is used when a statement is required syntactically but you do not want any command or code to execute.

pass
